# Prompt Records / 提示词留档

Use this reference for public examples, README/showcase covers, client work, or any cover likely to be reused.

Prompt records make Cover Pop auditable — especially when native Chinese titles, exact claims, and platform ratios are involved.

## Minimal Record

Save or report:

```text
asset name:
source / topic:
asset role:
platform + ratio:
title line(s) exact:
text ownership: native in-image / post-overlay (user-requested) / center-well no title
truth constraints:
archetype + aesthetic register:
final prompt:
output path:
rejected attempts:
QA result:
caption outside (if any):
```

## Rejected Attempts

Record a rejected attempt when it teaches a reusable lesson:

- garbled / melted Chinese title
- clickbait empty promise
- 微商风 / AI plastic / 土味
- unreadable at thumbnail size
- wrong platform ratio
- invented number or altered locked title
- wrong owner (should have been quote-cards / paper-operators / etc.)
- clutter / stacked archetypes

Do not keep noisy notes for every ordinary draft.

## File Placement

For this skill package:

- longer prompt records in `examples/` (e.g. a task-specific `PROMPTS.md`)
- showcase covers under `examples/` image folders when present

For client or article packages, use that project's asset folder.

## Why This Matters For Covers

- Native Chinese titles often need a second pass — record whether native render or user-requested overlay was used.
- Exact title strings and on-cover claims are easy to "improve" under pressure — the record catches drift.
- Platform ratio mistakes are silent until publish — recording ratio prevents repeat errors.
